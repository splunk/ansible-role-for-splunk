# Local testing harness

This directory contains a Docker + Molecule harness for exercising this role against systemd-based container topologies.

Goals:
- repeatable local integration testing for role changes
- realistic SSH connectivity (instead of docker connection for converge)
- optional web terminal into the controller for interactive debugging

## Prereqs
- Docker
- taskfile.dev (`task`)

## Quick start
```bash
cd testing

# one-time: secrets + images + downloads + network + runner
task setup

# create + prepare infra and run basic checks
task infra:test

# end-to-end (default env=small)
task workflow:full
```

## Environments
Select a topology via `MOLECULE_ENV`:
- `small` (default): 2 Splunk nodes + controller
- `dev`: 1 Splunk node + controller
- `prod`: 9 Splunk nodes + controller (+ git-server)

Examples:
```bash
MOLECULE_ENV=dev task infra:test
MOLECULE_ENV=prod task infra:test
```

## Web terminal (password-protected)
After `task infra:setup`, a local web terminal is exposed at `http://localhost:3000/ttyd`.

- Auth is enabled (ttyd `-c ansible:<password>`).
- The password is read from `testing/.secrets/ansible_password` (generated by `task setup:secrets`, not committed).

## Notes
- State (Docker volumes) is intentionally preserved across runs; use `task reset` if you want a destructive cleanup.
- See `testing/PROGRESS.md` for currently verified workflows.
